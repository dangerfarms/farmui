Why FarmUI

Everything you loved about Mantine, minus the runtime.

We kept the component API you know and rebuilt the styling layer on the modern CSS platform.

Semantic HTML

Real elements — <details>, <nav><ol>, role="tablist". The rendered markup is clean, meaningful and screen-reader friendly.

Zero runtime

Styles are static CSS in cascade layers. Nothing executes in the browser — fully compatible with React Server Components.

Accessible by default

WAI-ARIA patterns, focus management and full keyboard support are built into every component and tested against axe.

Modern CSS, everywhere

Container queries, color-mix(), light-dark(), logical properties and :where() — Baseline features, not polyfills.

Theme with variables

Rebrand by overriding --fui-primary. No provider, no theme objects — scope tokens to any subtree.

Tiny bundles

No styling engine ships to your users. Atomic, deduplicated, cacheable CSS and tree-shakeable components.

  • Zero dependencies
  • RSC-ready
  • No provider
  • RTL-ready
  • Respects reduced-motion
  • Tree-shakeable
  • TypeScript-first
  • No build plugin
See the difference

Same button. Cleaner everything.

FarmUI keeps styling in one shared, cacheable stylesheet — your markup stays readable. Utility frameworks inline dozens of classes onto every element; runtime CSS-in-JS serializes styles on each render.

Under the hood

Modern CSS, put to work.

No abstractions over the platform — FarmUI ships the same modern CSS features you’d reach for by hand, so nothing runs at runtime.

  • light-dark()

    One value, both themes — no duplicated theme objects.

    color: light-dark(#1a1a1a, #fafafa);
  • color-mix()

    Derive every hover and tint from a single token.

    color-mix(in oklch, var(--fui-primary) 12%, #0000)
  • @container

    Components respond to their container, not the viewport.

    @container (min-width: 20rem) { … }
  • :has()

    Style a parent from the state of its children.

    .field:has(:invalid) { border-color: red }
  • logical properties

    RTL-ready by default — no hardcoded left / right.

    padding-inline: 1rem; margin-block: 0.5rem;
  • @layer

    A predictable cascade — no specificity wars, no !important.

    @layer farmui.components { … }
  • clamp()

    Fluid type and spacing without a single media query.

    font-size: clamp(1rem, 2vw, 1.25rem);
  • oklch()

    Perceptually even, wide-gamut color you can reason about.

    --fui-primary: oklch(0.6 0.15 148);
Theming

Rebrand in one line of CSS.

No ThemeProvider, no theme objects. FarmUI is themed with CSS custom properties, so a single token change flows through every component.

  • Override --fui-primary to rebrand instantly
  • Scope tokens to any subtree for white-label apps
  • Native light & dark via light-dark()
/* Your brand, everywhere */:root {  --fui-primary: oklch(0.62 0.2 275);  --fui-radius-md: 0.75rem;  --fui-font-sans: "Satoshi";}
/* White-label a single section */.tenant-acme {  --fui-primary: oklch(0.65 0.18 25);}
